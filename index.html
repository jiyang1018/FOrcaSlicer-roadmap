Architecture & repo decisions
Design goals, hardware constraints, repo selection
▾
Design decisions locked
  • wall[0] = 0.2mm outer nozzle; wall[1+] = 0.4mm inner nozzle
  • Min 2 walls when same color AND material; 1 wall allowed otherwise
  • Slope-compensated gap between wall[0] and wall[1] only
  • Accumulator sync: 0.4 nozzle fires when next 0.2 layer pushes accumulated height above 0.4 max
  • Top/bottom: flat surfaces follow top/bottom settings; non-flat follows wall rules
  • Color-to-nozzle priority: color match → smallest nozzle = outer → extruder 0 fallback
  • Color depth: min = nozzle diameter; thin section = bleed-through or 50/50 split
  • Per-patch nozzle override: global=YES follows mixed logic; global=NO single nozzle owns patch
Decision change
  • Started on OrcaSlicer-FullSpectrum fork, later pivoted to Snapmaker Orca due to cleaner Filament for Features UI
  • FullSpectrum had resolved_model_config_for_tab stripping sparse_infill_filament — unfixable cleanly
Build environment
  • VS2019 required — build script hardcoded
  • CMake 3.29 works (repo requires 3.13–3.31)
  • Strawberry Perl in PATH causes CMake version conflict — fix: override PATH
  • Sentry cmake fix: changed Visual Studio 17 2022 → Visual Studio 16 2019
  • DLL strategy: copy all from installed Snapmaker_Orca\ except Snapmaker_Orca.dll
  • Build script: build_and_run.bat — builds, copies DLLs, launches snapmaker-orca.exe

Phase 4e
Per-nozzle line width scaling
PrintRegion::flow() rescales widths per extruder nozzle diameter
▾
Root cause
  • Line widths in fdm_process_U1.json were absolute values (0.42mm) sized for 0.4mm nozzle
  • When loaded as process preset, percentage is resolved against nozzle 1 before reaching flow()
  • config_width arrives at PrintRegion::flow() already as absolute, percent flag = false
Fix — PrintRegion.cpp
  • After config_width lookup, if has_mixed_nozzle_sizes and role is perimeter or infill:
  • Recover ratio: config_width.value / nozzle_diameter[0] (nozzle used during preset resolution)
  • Rescale: config_width.value = ratio * correct_extruder_nozzle_diameter
  • Applied to: frExternalPerimeter, frPerimeter, frInfill, frSolidInfill, frTopSolidInfill
fdm_process_U1.json
  • outer_wall_line_width → "105%"
  • inner_wall_line_width → "110%"
  • internal_solid_infill_line_width → "110%"
  • sparse_infill_line_width → "110%"
  • top_surface_line_width → "105%"
  • line_width → "110%"
  • support_line_width → "105%"
Milestone
  • Outer wall: 0.21mm (105% of 0.2mm nozzle) · Inner wall: 0.44mm (110% of 0.4mm) · Solid infill: 0.88mm (110% of 0.8mm)

Phase 5 — Color patch pipeline — May 17, 2026 🔄 In progress
Phase 5 — design
Color patch loop — correct architecture
Finalized design after failed approaches
▾
Goal
  • Paint face A red with filament 4 → only the outer N wall loops on face A print red
  • Inner loops and all infill remain base filament (continuous with rest of model)
  • N = color_patch_loops[filament_id - 1], configurable per filament
Failed approaches (do not retry)
  • ❌ ColorDepthProcessor 2D inward offset on segmented regions — offsets the wrong axis for vertical faces
  • ❌ PerimeterGenerator loop role overriding alone — painted region still creates a separate body with its own infill; red fills the interior
  • ❌ Fixed mm depth offset — conceptually wrong; depth should be in wall loops not mm
Correct architecture (not yet implemented)
  • multi_material_segmentation_by_painting assigns entire painted volume to painted filament — this creates the separate body problem
  • Correct fix: use painted region boundary only to identify which perimeter loops touch the painted face
  • In GCode.cpp ~line 4202 (the mixed nozzle perimeter split): extend to check loop adjacency against painted region boundary
  • Loops touching painted face boundary (outermost N) → painted filament; all other loops and all infill → base filament
  • Infill must always go to base filament — painted regions should NOT have their own infill body
What's already done
  • color_patch_loops (coInts) added to GCodeConfig in PrintConfig.hpp and PrintConfig.cpp
  • Added to init_filament_option_keys() so it resizes with extruder count
  • Default = 1 loop per filament
  • ColorDepthProcessor.hpp/.cpp preserved — may be useful for thin section detection later
Known limitations accepted
  • color_patch_loops is per-filament, not per-patch — all faces painted with same filament share loop count
  • Per-patch loop count deferred to future (requires FacetsAnnotation extension)
Next steps
  • In GCode.cpp process_layer(): for painted regions (wall_filament > 1), check each perimeter loop against the painted region boundary polygon at that layer
  • Loops that intersect boundary → painted filament for outer N loops, base for rest
  • Infill of painted regions → always route to base filament (extruder 1)
  • Add "Patch loops" spinner to painting gizmo panel (GLGizmoMmuSegmentation) after pen size
  • Wire spinner to color_patch_loops[selected_filament - 1] in config
