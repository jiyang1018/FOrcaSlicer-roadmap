Design decisions locked
  • wall[0] = 0.2mm outer nozzle; wall[1+] = 0.4mm inner nozzle
  • Min 2 walls when same color AND material; 1 wall allowed otherwise
  • Slope-compensated gap between wall[0] and wall[1] only
  • Accumulator sync: 0.4 nozzle fires when next 0.2 layer pushes accumulated height above 0.4 max
  • Color-to-nozzle priority: color match → smallest nozzle = outer → extruder 0 fallback
  • Color depth: min = nozzle diameter; thin section = bleed-through or 50/50 split
  • Per-patch nozzle override: global=YES follows mixed logic; global=NO single nozzle owns patch
Decision change
  • Started on OrcaSlicer-FullSpectrum fork, later pivoted to Snapmaker Orca due to cleaner Filament for Features UI
  • FullSpectrum had resolved_model_config_for_tab stripping sparse_infill_filament — unfixable cleanly
Build environment
  • VS2019 required · CMake 3.29 · Build script: build_and_run.bat
  • Strawberry Perl in PATH causes CMake version conflict — fix: override PATH
  • DLL strategy: copy all from installed Snapmaker_Orca\ except Snapmaker_Orca.dll

Phase 5
Color patch loop — convex surfaces working
Shell strip geometry correct for flat/convex faces · concave in progress
▾
Goal
  • Paint face A red (filament 4) → only outer N wall loops on face A print red; inner loops + infill = base filament
  • Overall model dimensions unchanged — red loops replace outermost N black loops spatially
  • N = color_patch_loops[filament_id - 1], configurable per filament, default = 1
Geometry — confirmed correct behavior
  • Convex outer face (cube): 1 red line on painted face only, white loops + infill fill interior ✅
  • Convex curved face (sphere): correct ✅
  • Corner behavior: outermost loop = full corner; inner loops retreat along angle bisector — falls out naturally from Clipper offset ✅
  • GCode confirmed: T3 prints 1 straight line per layer on painted face only ✅
Architecture implemented
  • Layer.hpp: color_patch_regions (vector<ExPolygons>) added — stores shell strip per extruder per layer
  • PrintObjectSlice.cpp apply_mm_segmentation(): stolen region shrunk to shell strip (N × nozzle_d × 1.2 wide); inner core returned to base region after all regions processed
  • Shell strip clipped to model surface boundary using parent_layer_region.raw_slices — removes interior segmentation cut edges
  • PerimeterGenerator.hpp/cpp: color_patch_regions pointer added; process_classic() limits loop_number to color_patch_loops when processing painted shell region
  • LayerRegion.cpp: color_patch_regions wired from layer to PerimeterGenerator
  • PrintConfig.cpp: color_patch_loops default value set (ConfigOptionInts(1,1))
Failed approaches — do not retry
  • ❌ ColorDepthProcessor 2D inward offset — wrong axis for vertical faces
  • ❌ PerimeterGenerator loop role overriding alone — painted region still creates body with its own infill
  • ❌ Boundary clipping via lslices surface strip — too thin, misses shell strip intersection
  • ❌ Boundary clipping via expanded/shrunk model_boundary — fails for concave (torus inner hole)
  • ❌ surface_adjacent (expand outside strip inward) — clips too aggressively
  • ❌ diff_ex(shell_strip, model_interior) — interior cut sides still survive near boundary
Known issue — concave surfaces
  • Torus inner face (concave): red still shows as wedge outline instead of single line
  • Root cause: raw_slices for base region at concave layers may already exclude the hole boundary — investigating whether raw_slices is populated before apply_mm_segmentation runs
  • Next: verify raw_slices content at mid-height torus layers; if empty, alternative source needed for pre-segmentation geometry

已锁定的设计决策
  • wall[0] = 0.2mm 外墙挤出口；wall[1+] = 0.4mm 内壁挤出口
  • 相同颜色且相同材料时，最少 2 条总墙壁；否则允许 1 条
  • 仅在 wall[0] 和 wall[1] 之间进行坡度补偿间隙
  • 累加器同步：当 0.2 层累计高度超过 0.4 挤出口最大层高时触发 0.4 挤出口
  • 颜色到挤出口优先级：颜色匹配 → 最小挤出口=外墙 → 挤出机 0 兜底
  • 颜色深度：最小值=挤出口直径；薄截面=渗透或 50/50 分割
  • 每区域挤出口覆盖：global=YES 遵循混合逻辑；global=NO 单挤出口拥有整个区域
决策变更
  • 最初基于 OrcaSlicer-FullSpectrum 分支，后因 Filament for Features UI 更清晰改用 Snapmaker Orca
  • FullSpectrum 存在 resolved_model_config_for_tab 过滤 sparse_infill_filament 的问题，无法干净修复
构建环境
  • 需要 VS2019 · CMake 3.29 · 创建脚本：build_and_run.bat
  • 系统路径中的 Strawberry Perl 会导致 CMake 版本冲突 — 修复：覆盖系统路径

目标

  • N = color_patch_loops[filament_id - 1]，按耗材可配置，默认值 = 1
几何 — 已验证正确行为
  • 凸面外表面（立方体）：涂色面仅 1 条红线，白色循环 + 填充填满内部 ✅
  • 凸曲面（球体）：正确 ✅
  • 转角行为：最外层循环 = 完整转角；内层循环沿角平分线退缩 — 由 Clipper 偏移自然产生 ✅
  • GCode 已验证：T3 每层仅在涂色面打印 1 条直线 ✅
已实现架构
  • Layer.hpp：新增 color_patch_regions (vector<ExPolygons>) — 按挤出口按层存储壳带
  • PrintObjectSlice.cpp apply_mm_segmentation()：stolen 区域缩减为壳带（宽度 = N × nozzle_d × 1.2）；所有区域处理完成后将内核归还至基础区域
  • 壳带使用 parent_layer_region.raw_slices 裁剪至模型表面边界 — 去除分割产生的内部截面边
  • PerimeterGenerator.hpp/cpp：添加 color_patch_regions 指针；process_classic() 处理着色壳区域时将 loop_number 限制为 color_patch_loops
  • LayerRegion.cpp：color_patch_regions 从 layer 传入 PerimeterGenerator
  • PrintConfig.cpp：color_patch_loops 默认值已设置（ConfigOptionInts(1,1)）
